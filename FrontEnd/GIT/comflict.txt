РЕШЕНИЕ КОНФЛИКТА 

(1) git branch - смотрим какие ветки есть
(2) git checkout -b (название ветки) - создаем новую ветку
(3) внести необх. правки в новой ветке (через любимый редактор)
(4) выполнить индексацию и фиксацию правок (git add & git commit или git commit -am 'комментарии')
(5) переключиться на ветку master (git checkout master)
(6) вносим правки в файле в том же месте в новой ветке (через любимый редактор)
(7) выполнить индексацию и фиксацию правок (git add & git commit или git commit -am 'комментарии')
(8) попробовать сделать merge или 
			сделать git merge --abort (отменить слияние)
(9) после выпадания ошибки удалить ненужную строку вручную через любой редактор
                             или оставить оба варианта (Accept both)
(10) сделать git add . в master
(11) сделать git commit -m 'комментарии' в master
